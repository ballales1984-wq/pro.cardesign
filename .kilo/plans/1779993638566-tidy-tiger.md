## Test Failures (from `node tests/test_coverage.js`)
### ES Module Import Issues (Unexpected token '{')
- Affects: Voxel local properties, VoxelModel import, HybridModel import, GPUCompute import, GPUCompute worker tests.
- Likely cause: The custom test script uses dynamic `import()` to load ES modules, but Node.js treats `.js` files as CommonJS unless `"type":"module"` is set in the nearest `package.json`. The project root lacks this flag, causing import/export statements to be syntax errors.
- Note: Some tests pass (MaterialSystem, ModuleSystem, Brick) because they either lack import statements or have only exports that may be tolerated? Actually, exports also cause syntax errors in CommonJS. The passing tests suggest the environment may treat certain files as ES modules due to unclear resolution.
