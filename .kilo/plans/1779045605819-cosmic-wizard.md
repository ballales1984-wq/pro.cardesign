## Fase 1 — Bug Critico: ScalingTool NaN (scaling-tool.js:162)

### Causa radice
```js
// RIGA 162 — scaling-tool.js
const delta = intersection[this.dragAxis] - this.dragStartPoint[this.dragAxis];
```
- `intersection` è un `THREE.Vector3`, non un array. L'accesso `intersection['x']` (o `[0]`) **non** restituisce le coordinate ma `undefined`.
- Il risultato è `NaN - NaN = NaN`. Tutte le operazioni di scala successive (es. `_updateScaleLabel`) ricevono `NaN`.

### Fix proposto
Sostituire l'accesso con dot-notation esplicito:
```js
const p = intersection;
const dp = this.dragStartPoint;
const delta = (this.dragAxis === 'x' ? p.x : this.dragAxis === 'y' ? p.y : p.z)
            - (this.dragAxis === 'x' ? dp.x : this.dragAxis === 'y' ? dp.y : dp.z);
```

## Fase 2 — Bug Media: Map usato come array (voxel-engine.js:625)

### Causa radice
```js
// RIGA 625–626 — voxel-engine.js (_removeVoxelSilently)
idxMap[instanceId] = null;   // idxMap è un Map, non un array!
```
`instanceToKey` è dichiarato come `new Map()` alla riga 27. L'accesso `idxMap[instanceId]` aggiunge una proprietà arbitraria al **prototipo dell'istanza Map**, senza restituirla via `.get()`. Di conseguenza la voce non viene mai cancellata e la Map si riempie di entry fantasma.

### Fix proposto
Uso della API corretta di `Map`:
```js
// idxMap.delete(worldKey)  ← da aggiungere
idxMap.delete(worldKey);
```

## Fase 3 — Bug Bassa: HTML invalido (index.html:183)

### Fix proposto
Sostituire il tag auto-chiudente con tag di apertura/chiusura standard:

```html
<!-- DOPO (valido) -->
<div id="modules-tree" aria-label="Albero moduli"></div>
```
