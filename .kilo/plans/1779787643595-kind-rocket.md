# Logical Errors Analysis - pro.cardesign

## Critical Issues Found

### 1. Missing Center Tracking in Scale History (CRITICAL)
**File:** `src/voxel-engine.js` lines 886-894, 936-945

The undo/redo code references `action.oldCenter` and `action.newCenter` properties that are never pushed to history:

- Line 363-370: `_pushHistory` pushes `{ type: 'scale', x, y, z, oldScale, newScale }` — NO center data
- Lines 890-891, 898-899, 933-934, 941-942: `undo/redo` tries to read `action.oldCenter`/`action.newCenter` — **UNDEFINED**

**Impact:** Scale undo/redo silently fails because center properties are undefined.

### 2. Inconsistent `_setInstanceMatrix` Signature
**File:** `head-ve.js` vs `src/voxel-engine.js`

- `src/voxel-engine.js:241` has: `_setInstanceMatrix(mesh, instanceId, position, scale = new THREE.Vector3(1, 1, 1))`
- `head-ve.js:156-159` calls `matrix.makeScale(scale.x, scale.y, scale.z)` **without** normalizing scale to Vector3

**Impact:** Potential crashes if scale is passed as an array instead of Vector3 object.

### 3. Inconsistent `_removeVoxelSilently` Implementation
**File:** `head-ve.js:647` vs `src/voxel-engine.js:759`

- `head-ve.js:647`: `idxMap.delete(worldKey)` — removes from Map
- `src/voxel-engine.js:759`: `idxMap[instanceId] = null` — array assignment

**Impact:** Wrong data structure being modified (Map vs Array inconsistency).

### 4. Missing Property Initialization in head-ve.js
**File:** `head-ve.js`

Missing `this.voxelSize = 1.0` and `this.cameraNavigationMode = false` — used by tools but never initialized.

### 5. Duplicate Test in test_coverage.js
**File:** `tests/test_coverage.js` lines 576-592

Redundant test already covered at lines 420-426.
