# Development Plan for pro.cardesign

## Overview
This plan outlines the development steps for the pro.cardesign application, aligning with the established priorities in AGENTS.md. The application focuses on a brick-based design system with Python core logic and JavaScript/Three.js rendering.

## Current Status
Based on AGENTS.md, the project has:
- Python core (`core/`) with brick dataclass and component definitions
- JavaScript rendering (`src/`) with voxel engine, material system, and UI
- Established conventions for file naming, brick sizing, and JSON serialization
- Defined development priorities and testing requirements

## Development Priorities & Implementation Plan

### 1. Brick System — size, position, overlap
**Goal:** Implement robust brick manipulation with independent dimensions and collision detection.

**Tasks:**
- [ ] Verify brick.py implements proper mm-based positioning and sizing
- [ ] Implement overlap detection algorithm in core/ component.py
- [ ] Create unit tests for brick placement and collision scenarios
- [ ] Update voxel-engine.js to use brick system for rendering
- [ ] Test: python -m pytest tests/test_coverage.py -v

### 2. Scaling Tool — drag face, live dimensions
**Goal:** Enable interactive scaling of bricks via face dragging with real-time dimension feedback.

**Tasks:**
- [ ] Examine src/core/scaling-tool.js for current implementation
- [ ] Implement face detection for drag initiation
- [ ] Add live dimension display during scaling operations
- [ ] Ensure scaling respects voxel grid constraints
- [ ] Test: Manual verification of scaling tool functionality

### 3. Component Library — parametric, wheels, tubes
**Goal:** Create reusable parametric components beyond basic bricks.

**Tasks:**
- [ ] Extend component.py with parametric component definitions
- [ ] Implement wheel and tube component factories
- [ ] Create UI integration in src/core/component-library.js
- [ ] Add tests for component instantiation and parameter validation
- [ ] Test: python -m pytest tests/test_coverage.py --cov=core

### 4. Project Save/Load — scale included in JSON
**Goal:** Ensure project serialization includes voxel scale and supports full restoration.

**Tasks:**
- [ ] Verify toJSON()/fromJSON() methods in voxel-engine.js include scale
- [ ] Test save/load cycle preserves all brick properties and scaling
- [ ] Implement versioning in JSON format for future compatibility
- [ ] Test: node tests/test_coverage.js for JavaScript save/load

### 5. Import STL + Quality — ovality, deviation, import
**Goal:** Enable STL import with quality analysis for ovality and deviation metrics.

**Tasks:**
- [ ] Review src/core/stl-import.js STL parser implementation
- [ ] Integrate QualityAnalyzer for ovality/deviation calculation
- [ ] Add import UI components and error handling
- [ ] Create test STL files with known defects for validation
- [ ] Test: Manual import of various STL files and quality reporting

## Development Process
1. Each priority must have corresponding tests before implementation completes
2. Follow existing code conventions (kebab-case JS, snake_case Python)
3. Maintain separation of geometry (Python) and rendering (JavaScript)
4. Run full test suite after each major change
5. Commit only after tests pass

## Dependencies
- No new dependencies to be added without discussion
- Use existing testing frameworks (Python pytest, Node.js test)
- Leverage existing material system and physics calculations

## Timeline
Work in priority order, but parallel work possible on independent subsystems.

## Definition of Done
- All tests pass for the implemented feature
- Code follows project conventions
- Documentation updated if necessary
- Changes committed to Git
